Project: Create a Python Telegram Bot for an AI-Driven Rice Field Monitoring System.

Objective: Develop a bot that acts as an information and alert interface for farmers. Users should be able to ask questions about their field's status (e.g., weed alerts, crop health) and receive objective, data-driven answers. The bot should also be capable of pushing automated alerts (like the ones generated by our Jetson Nano system) to subscribed users.

Core Features to Implement:

Command Handler:

/start & /help: Welcome message and list of commands.

/subscribe: Allows a user to subscribe to automated alerts for a specific field (for future use; can simulate with a user list for now).

/unsubscribe: Allows a user to unsubscribe from alerts.

Query Handler (Primary Function):

The bot should listen for messages and use a simple keyword-matching logic to provide pre-defined, objective answers based on the project's capabilities.

Example user queries and intended bot responses:

User: "Weed alert in my field?"

Bot: "Our latest scan has detected weeds in Sector B-5. GPS: [LINK_TO_GMAPS]. Recommended action: Targeted herbicide application. Type 'details' for more."

User: "How is my crop health?"

Bot: "Overall NDVI health index is 0.72, indicating good vigor. A minor nutrient deficiency was detected in the northern plot. A detailed report has been generated on your web dashboard."

User: "What is the yield prediction?"

Bot: "Based on current health and historical data, the predicted yield for your rice field (var. Basmati) is approximately 6.2 tons/hectare."

User: "What actions should I take?"

Bot: "1. Inspect weed presence at Sector B-5. 2. Consider soil amendment for nitrogen in the northern plot. Re-scan scheduled in 15 days to monitor progress."
